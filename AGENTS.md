# Ideal customer profile (ICP)

This file is intentionally **gitignored**—local strategy notes for founders/operators. The tracked contributor entry point remains [`AGENTS.md`](./AGENTS.md).

## Primary ICP

**Founding team / startup operator** — small teams (often 2–15 people) where a founder or operator needs to run the business with AI helpers **without** losing structure: goals, assigned work, light governance, and a clear trail of decisions.

## Positioning (one line)

Bopo is for founding teams who run AI like a small company—clear goals, assigned work, and explicit approvals—without losing the thread in chat threads.

## Hero workflow (north star)

1. Define goals for the week or initiative.
2. Break work into issues and assign an agent (or human) where it helps.
3. Use heartbeats / loops for recurring check-ins on the same work.
4. Route anything sensitive through approvals so operators stay in control.

## Product implications

- Lead with **outcomes and oversight**; treat repo/workspace/adapter depth as **advanced** paths.
- Measure **time-to-first useful heartbeat or loop**, not time-to-perfect git setup.
- Champion may still be technical early on; onboarding should not **require** SRE-level setup for the operator story to feel true.

## Explicitly not (for now)

- Enterprise IT buyers needing SSO, compliance packs, and procurement-heavy rollout on day one—unless you choose to build that lane separately.
- Teams that want a single generic chat with no notion of company, projects, or approvals.
